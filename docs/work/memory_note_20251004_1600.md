**Performance**:

- Long audio (29min): 29% speedup (208.0s vs 267.5s)
- Longer audio = better speedup (more chunks to parallelize)

**Why 3 threads optimal**:
- 2 GPU threads: 11.14s (SLOWER - 1.26x)
- 3 GPU threads: 7.82s (FASTER - 1.13x) ← WINNER
- 4 GPU threads: 8.94s (SLOWER - 0.99x)
- Apple M1 has ~2 effective GPU queues, 3 workers hits the sweet spot
- More workers = batch decode overhead increases (1.57ms → 2.81ms → 3.68ms per run)
